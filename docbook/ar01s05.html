Program Options

Note

I've noticed in the past that it can be really hard to keep this section up-to-date, so bear that in mind.

--help

Display help information.

--version

Displays version information.

--base_dir=<BASE_DIR>

base directory of path for outputted files. Filenames will be relative to the provided base directory, or the current working directory if none is provided.

--log_file=<LOG_FILE>

Name/path-name of generated log file. Will be relative to LOG_DIR if its provided, or relative to BASE_DIR if LOG_DIR isn't provided.

--log_dir=<LOG_DIR>

Base directory for the log file. This takes precedence over BASE_DIR for the log file. Provided so that the log file can be outputted in a special directory (e.g: put it in /var/log).

--log_level

The log level. Filters what is written to the log file.

Critical/Fatal
Error
Warning
Info
Debug

Lower levels include more information. Choosing a level outputs the information of that level and all above it.

--template

Specifies the template file relative to BASE_DIR. Template file acts as a template for the output. The template is a python string that will have format() called on it. Valid identifiers include:

{upgrade}
{install}
{remove}
{not_upgraded}
{time}
{upgradeable}

The first 4 are taken from apt-get. time is current time (see --time_format). upgradeable is the sum of upgrade and not_upgraded, i.e: the number of packages that could be upgraded, which is what most will want to use.

--time_format=<TIME_FORMAT>

Defines the format of the time placeholder {time} (see --template). Uses python time formatting syntax (which is similar if not the same as the c syntax.)

--max_width=<MAX_WIDTH>

Max width of the output.

--no_error_output

If an error occurs, do not write the output file.

-c / --num_update_checks =<NUM_UPDATE_CHECKS>

Number of time to try updating (apt-get update). Useful in situations where apt-get update randomly fails every once in a while.

--sleep_period=<SLEEP_PERIOD>

Time (in seconds) to sleep between update attempts. Useful when update fails are temporally localized. Values less than or equal to 0 mean no sleeping.

--no_update

Don't update. Useful when your computer is already updating frequently enough. Also useful for debugging/testing. Conflicts with --num_update_checks. If both are used, then the behavior is undefined.

--no_root

Don't do anything requiring root privileges. Allows normal users to use this program. Things requiring root privileges (e.g: updating) can be done in the root crontab.

--network_check

Enable the network check, which tries to determine whether the network is up and available before trying to update. Rather inaccurate and not very useful now that we have --num_udpate_checks.

Note

I'm contemplating depreciating/removing this option as its not very effective.
